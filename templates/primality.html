Prime number

Prime numbers are natural numbers greater than 1 that have exactly two factors: 1 and the number itself. This means that a prime number is only divisible by 1 and itself. For example, the number 5 is a prime number because it has only two factors, 1 and 5. On the other hand, the number 6 is not a prime number because it has more than two factors: 1, 2, 3, and 6

Primality testing is the act of determining whether a given number is prime. Several algorithms have been developed to test primality, including the Fermat test, the Miller-Rabin test, and the AKS test. These algorithms use different mathematical properties to determine whether a number is prime or composite. Primality testing is an important concept in number theory and cryptography, as prime numbers are used in many cryptographic algorithms.

Fermat Test

The Fermat primality test is a probabilistic algorithm used to determine whether a given number is prime or composite. The test is based on Fermat's Little Theorem, which states that if p is prime and a is not divisible by p, then a(p-1) ≡ 1 (mod p). To test whether p is prime, one can pick random integers a not divisible by p and see whether the congruence holds. If it does not hold for a value of a, then p is composite.

Miller-Rabin

The property is the following. For a given odd integer n > 2, let’s write n − 1 as 2sd where s is a positive integer and d is an odd positive integer. Let’s consider an integer a, called a base, which is coprime to n. Then, n is said to be a strong probable prime to base a if one of these congruence relations holds:

  • ad ≡ 1 (mod n)
  • a2rd ≡ -1 (mod n) for some r such that 0 ≤ r < s

AKS (Agrawal - Kayal - Saxena)

This algorithm is significant because it is the first to achieve general, polynomial-time, deterministic, and unconditionally correct primality testing. The AKS primality test is based on the following theorem: An integer n greater than 2 is prime if and only if the polynomial congruence relation holds for some a coprime to n. The algorithm evaluates this congruence for a large set of a values, whose size is polynomial to the digits of n.
